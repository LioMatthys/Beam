What it is

See and control a real phone from your computer.

Install one app, flip on Accessibility once, and your Android device is drivable from your machine over Wi-Fi: read what's on screen, tap by element, type, swipe, scroll. The live screen mirrors to your PC as the agent's eyes. Every other way to do this (Appium, UIAutomator, Maestro, most "mobile-use" agent stacks) needs ADB, USB debugging, or an emulator. Beam needs none of it.

How it works

One wire protocol, three actors.

Technology

Built on hardware paths, not hacks.

Android sender — Expo / React Native UI + a Kotlin native module: MediaProjection + MediaCodec (hardware H.264), AccessibilityService for reading & input.
Windows receiver — Electron + React + TypeScript. H.264 decode via the browser-native WebCodecs (hardware-accelerated) to a canvas.
Wire protocol — TCP, length-prefixed multiplexed frames (video + control). Self-signed TLS via AndroidKeyStore with trust-on-first-use cert pinning.
Agent layer — DroidPilot (Python) exposes the control ops as an MCP tool surface; record/replay flows with pass/fail reports.

Quick start

Three one-time steps, then connect.

1

Install the Windows app

Grab Beam-Setup.exe (or the portable zip) from the releases page and run it.

2

Install the phone app & enable Accessibility

Scan the QR in the Windows app to install Beam.apk, then turn on Settings → Accessibility → Beam control once.

3

Connect

Tap Start sharing on the phone, enter its IP + 6-digit code on the PC (or scan to pair). The live screen appears; it auto-reconnects.

4

(Optional) point an agent at it

The Windows app serves a localhost JSON-RPC on 127.0.0.1:8788. Connect your agent, or use DroidPilot as an MCP.

Enabling control (one-time)

Casting needs nothing extra. To let the desktop (or an AI agent) control the phone, switch on Beam's Accessibility service once. Because Beam is sideloaded, Android hides the toggle for security at first — here's the 2-step unlock:

No-friction shortcut: install from the Windows app's ☰ menu → "Install via USB" (phone plugged in) and the toggle is not greyed out — skip step 1 entirely.

Honest limits. Beam is built for trusted Wi-Fi: the link is TLS-encrypted with cert pinning, but it's LAN-only and the pairing code is the authorizer (full PAKE + remote relay are roadmap). Distributed by sideload / MDM, since Accessibility automation is restricted on the Play Store. Games and some WebView/canvas surfaces expose a thin element tree — the screenshot op is the vision fallback there.
